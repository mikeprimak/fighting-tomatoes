## Part 1 — Crash fix rollout status

### Done
- Root cause identified: OAuth URL scheme in `app.json` pointed at Google project `good-fights-app` (`499367908516-...`) while `useGoogleAuth.ts` pointed at a different project `fight-app-ba5cd` (`1082468109842-...`). iOS Google Sign-In SDK hard-asserts on URL scheme/client-ID mismatch, causing a native crash JS can't catch.
- Mobile code updated: both `GOOGLE_CLIENT_ID_WEB` and `GOOGLE_CLIENT_ID_IOS` in `packages/mobile/hooks/useGoogleAuth.ts` now point at `good-fights-app`.
- Backend code updated: `GOOGLE_CLIENT_ID` env var now parsed as optional comma-separated list of accepted audiences in both `packages/backend/src/controllers/authController.ts` and `packages/backend/src/routes/auth.fastify.ts`. Uses helper `getGoogleAudiences()`.
- Render env var `GOOGLE_CLIENT_ID` set by user to:

  (new audience first, legacy audience second — keeps old Android builds working during rollout). Backend auto-redeployed.

### Key decisions locked in during this session

- **Official slogan: "Never miss a good fight."** (exact casing — only "Never" capitalized, lowercase "good fight", period at end). Saved to memory at `project_good_fights_slogan.md`. Use verbatim in ALL marketing copy going forward.
- **User voice**: short, punchy, colloquial, uses fight-fan vernacular ("wanna watch", "fired up for", "fans loved"). Avoid corporate marketing-speak. When in doubt, lean short and specific over long and abstract.
- **Feature set included in description**: schedule, hype scores, walkout notifications, fight ratings, comments. **Intentionally excluded** from description: spoiler-free mode, follow fighters, live results, predictions. User made this call — don't add them back without asking.
- **"15+ promotions and growing"** — user confirmed there are ~15 promotions, and the "and growing" matters for positioning. The named orgs in the description are examples, not exhaustive.
- **Comments subline was workshopped heavily.** User rejected permanence angle ("they stay up forever"). Picked "The comment section combat sports deserves." from a set of 5 options. Don't revisit this — it's locked.

### Next tasks in the App Store optimization sequence

Session ended right before subtitle work. Suggested order:

1. **Subtitle** — 30-character limit, appears right under app name. High-impact. Needs to complement the slogan/description without duplicating. Marketing plan suggests keyword terms like "UFC schedule", "MMA fights", "boxing schedule", "fight notifications", "combat sports" — subtitle is one of the weakest keyword signals, but fitting a search term here is still useful.

2. **Keywords** (100 characters) — the strong iOS search-ranking signal. Per marketing plan, should include: "UFC schedule, MMA fights, boxing schedule, fight notifications, combat sports". Comma-separated, no spaces after commas to save characters. Don't repeat words already in app name/subtitle — Apple indexes those separately.

3. **Screenshots** — Marketing plan's ideal order: (1) schedule view, (2) notifications feature, (3) ratings view. User was planning to shoot a preview video (beats 1-4 captured per `docs/daily/2026-04-18.md` — still in CapCut-editing stage).

4. **Promotional Text** (170 chars, editable without resubmission) — useful for fight-week hype / campaign-specific messaging. **Evergreen copy locked:** `Every UFC, boxing, PFL, ONE & BKFC card in one schedule. Hype scores. Walkout alerts. Fight ratings. Built by a fan, for fans.` (128 chars). Swap for fight-week variants when UFC 328 / MVP Netflix / UFC White House approach.

5. **App Store Preview video** — user has raw captures but hasn't finished editing. Blockers documented in `docs/daily/2026-04-18.md`.
