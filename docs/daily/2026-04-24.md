## Changes Made

### Pull-to-refresh on all main tabs
Added a `RefreshControl` to every main tab screen. Each refresh re-runs the tab's primary data query; React Query's `refetch()` is awaited so the spinner stays visible until the network round-trip completes.

Files touched:
- `packages/mobile/app/(tabs)/live-events.tsx` — FlatList, wraps the `['upcomingEvents', ...]` infinite query
- `packages/mobile/app/(tabs)/events/index.tsx` — FlatList, same `['upcomingEvents', ...]` query
- `packages/mobile/app/(tabs)/past-events/index.tsx` — FlatList, `['pastEvents', ...]` query
- `packages/mobile/app/(tabs)/top-fights/index.tsx` — FlatList, `['topRecentFights', ...]` query
- `packages/mobile/app/(tabs)/profile.tsx` — ScrollView; refresh fans out to `refreshUserData`, `getPredictionAccuracyByEvent`, `getGlobalStanding`, `getMyTopReviews`, `getMyTopPreflightComments`

All use the same shape: `const [isRefreshing, setIsRefreshing] = useState(false)` + an async `onRefresh` callback that awaits the refetch. `tintColor`/`colors` both set to `colors.primary` so the spinner matches the brand yellow.

### Live events periodic self-update
Live Events already polled every 30s via `refetchInterval: 30000` on the `['upcomingEvents', ...]` infinite query — it was working before. Added `refetchIntervalInBackground: false` explicitly so we don't burn requests when the app is backgrounded (default behavior anyway, but now it's intentional).
