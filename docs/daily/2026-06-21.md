## Home event band: "No events this weekend" / web empty on Sundays

**Symptom:** Mobile home "This Weekend" section showed "No events this weekend";
web `WeekendEventsSection` rendered nothing at all.

**Root cause:** Both surfaces shared the same window — "today up to the Monday
that starts next week" (rest of the current Mon–Sun calendar week). On a **Sunday**
that collapses to a single day:
`daysUntilNextMonday = (1 - 0 + 7) % 7 = 1` → window = `[Sun, Mon)` = just today.
Cards cluster Fri/Sat, so by Sunday the weekend is over and the next card is
outside the one-day window. Today (Sun Jun 21) had no events; soonest was BKFC
Fri Jun 26. Mobile shows the empty state; web returns `null` when the window is
empty.

**Fix:** Replaced the week-bounded window with a rolling **7-day** window
(`todayKey + 7*DAY_MS`, today + next 6 days) in:
- `packages/mobile/app/(tabs)/home.tsx`
- `packages/web/src/components/home/WeekendEventsSection.tsx`

Also renamed misleading placeholder copy on mobile: "This Weekend" → "This Week",
"No events this weekend" → "No events in the next 7 days". Per-day headings
("Events Today/Tomorrow/<weekday>") were already day-derived and unchanged.

**Verification:** Web `tsc --noEmit` clean. Window now surfaces Jun 26 (BKFC, ONE)
and Jun 27 (UFC Fiziev vs. Torres, ONE, PFL).

## Mobile blog rich blocks (FB embeds + native fight cards) + Conor article embeds

Mike: add Facebook embeds to the "Every Conor McGregor Fight, Ranked" article;
also FB embeds were too wide on mobile and a web-only fix had left them invisible
on mobile. Commit `59cfce7f` → `main`.

**Root cause / architecture finding.** Rich blocks in posts are raw HTML divs the
WEB hydrates into components (`fb-post` → FB SDK embed via `FacebookEmbeds.tsx`;
`gf-fight-card` → live React card via `BlogFightCards.tsx`). The native reader
(`app/blog/[slug].tsx`, `react-native-render-html`) can't run that hydration, so
**both embeds AND interactive fight cards rendered as empty/invisible** on mobile.
The "fix" commits (`f3b3d2e2`, `4916cd5b`) only touched web. Discussed framework
options with Mike (markdown vs MDX vs CMS) — **kept markdown**; the real debt was
the rendering contract, not the storage. MDX is a web-only dead end for RN; a CMS
is over-engineering at ~20 self-authored posts.

**Built the mobile rich-block layer** (`components/blog/BlogRichBlocks.tsx`):
- `transformBlogBlocks()` rewrites the empty web divs → custom `<fbembed>` /
  `<gffight>` block tags (registered via `customHTMLElementModels`) so the render
  engine doesn't prune them. Verified against real markdown-it output: 12 fbembed
  + 15 gffight produced, 0 leftovers, 0 failed matches.
- FB embeds render in a **WebView** (`react-native-webview@13.15.0`, newly added),
  width-clamped to the column (FB plugin 350px floor, 500 cap), auto-height via
  injected `scrollHeight` postMessage, **lazy-mounted** when scrolled within 700px
  (measureLayout against a single content wrapper; 5s fallback so content is never
  lost).
- `gf-fight-card` → native spoiler-neutral card (fetches `getFight`, taps to
  `/fight/[id]`), mirroring web `BlogFightCard`. Fixes fight cards on mobile too.

**⚠️ Needs a new native build** — `react-native-webview` is a native module; an OTA
can NOT deliver it. Do NOT `eas update` this mobile code until a build with the
module ships, or the blog screen will crash on `require`. Mike pre-approved a build
for this; not started yet (ask before running `eas build`).

**Article embeds** (`...conor-mcgregor-fights-ranked.md`): 12 FB embeds total (2
originals kept: Brimage, Diaz 2). Added/swapped 10: Floyd (replaced static photo),
Cerrone, Siver, Brandao, Poirier 1, Aldo, Poirier 3, Mendes, Alvarez (swapped its
old embed), Khabib. Static Wikimedia headshots replaced where an embed was given;
