# 2026-05-28

## Fighter records backfill (0-0-0 problem) + UI guard

Mike: "many of my fighters show 0-0 / 0-0-0 ... can we get accurate records
today? scrape ufcstats.com?" — yes. Two-part fix, both shipped.

### Scale of the problem (before)

- **9,718** total fighters; **8,241 (84.8%)** at the 0-0-0 default.
- **6,800** of those empties actually appear on ≥1 card (real UX impact).
- By sport: MMA 6,904, Boxing 1,190, Wrestling 71, Muay Thai 62, Kickboxing 14.
- Cause: `wins/losses/draws` are Int columns defaulting to 0; only the active
  UFC roster gets filled by the daily ufc.com scrape. Everyone else (historic
  UFC, opponents, other orgs) stays 0-0-0.

### Part 1 — data backfill from ufcstats.com

- **New gotcha:** ufcstats.com now gates every response behind a lightweight
  Hashcash-style JS proof-of-work ("Checking your browser…"). Page embeds a
  `nonce` + difficulty (2 leading hex zeros); client finds `n` where
  `sha256(nonce:n)` starts with `00`, POSTs `nonce`+`n` to `/__c`, gets a
  clearance cookie. Solved in Node with built-in `crypto` — **no browser**.
  (The old "ufcstats is not protected" note is now stale.)
- `packages/backend/src/services/scrapeUFCStatsFighters.ts` — pulls the 26
  letter pages of the fighter directory (`/statistics/fighters?char=<a>&page=all`),
  parses name + W/L/D. 4,495 fighters total on ufcstats.
- `packages/backend/src/scripts/backfillFighterRecords.ts` — dry-run by default,
  `--apply` to write. Matches our 0-0-0 MMA fighters by normalized full name,
  **conservatively**: skips anything ambiguous on either side, never overwrites
  a populated record. Writes `prisma/fighter-records-backfill.json` audit log.
- **Result: 2,676 MMA records filled → empty share 84.8% → 57.3%** (MMA empties
  6,904 → 4,228). Sample sanity: Court McGee 23-14-0, Wilson Reis 23-10-0,
  CM Punk 0-1-0 — all correct.

**Coverage reality:** ufcstats is UFC/Zuffa-only. The 4,216 unmatched MMA
fighters are genuinely not in ufcstats (regional/non-UFC). The **1,190 boxers**
are entirely out of scope — boxing needs a different source (BoxRec is heavily
bot-protected). Mike: "ufc only is ok" → boxing deferred.
