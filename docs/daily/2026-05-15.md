## Session 2 — Hype Reveal Modal SHIPPED

Picked the work back up. Three fixes closed out the feature.

### Width-parity bug — root cause + fix

Skipped the `onLayout` instrumentation plan; structural diff was visible on read. Both `kavContainer`s wrap their `modalContainer` (which has `width: '88%'`). The reveal's `kavContainer` had `width: '100%'`, the hype's did not. The hype modal's overlay has `alignItems: 'center'`, which means a child without an explicit width shrinks to content rather than stretching. So the hype modal's KAV was narrower than the reveal's KAV, and `'88%'` of two different parents = two different pixel widths.

Fix: one line, added `width: '100%'` to the hype modal's `kavContainer` (UpcomingFightModal.tsx:709). Confirmed on Pixel.

### Close button width — match the Done button

Reveal's Close button was `paddingHorizontal: 56`, fine for shape but didn't match the Done button's horizontal extent. Done lives in a row with a 46pt notify bell + 10pt gap + 8pt padding each side, so Done width = (scrollContent inner) − 72. Switched Close to `alignSelf: 'stretch'` + `marginHorizontal: 36`, which produces the same final pixel width centered.

### Glow — from 3-layer halo to smooth Gaussian-style fade

Mike saw the prior glow as three distinct concentric borders rather than a soft shadow. Replaced the hand-tuned `haloInner/Mid/Outer` with `GLOW_LAYERS`, a generated array of 14 layers each 1px further out with exponential opacity decay (peak ~0.32 at the bar edge, decaying through `0.32 * exp(-i/4.2)`). Visually merges into a continuous fade on both iOS and Android. iOS `shadowColor`/`shadowRadius` kept on the bar itself for the extra blur on iOS.
