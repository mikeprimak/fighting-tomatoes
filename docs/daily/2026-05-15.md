## Session 2 — Hype Reveal Modal SHIPPED

Picked the work back up. Three fixes closed out the feature.

### Width-parity bug — root cause + fix

Skipped the `onLayout` instrumentation plan; structural diff was visible on read. Both `kavContainer`s wrap their `modalContainer` (which has `width: '88%'`). The reveal's `kavContainer` had `width: '100%'`, the hype's did not. The hype modal's overlay has `alignItems: 'center'`, which means a child without an explicit width shrinks to content rather than stretching. So the hype modal's KAV was narrower than the reveal's KAV, and `'88%'` of two different parents = two different pixel widths.

Fix: one line, added `width: '100%'` to the hype modal's `kavContainer` (UpcomingFightModal.tsx:709). Confirmed on Pixel.

### Close button width — match the Done button

Reveal's Close button was `paddingHorizontal: 56`, fine for shape but didn't match the Done button's horizontal extent. Done lives in a row with a 46pt notify bell + 10pt gap + 8pt padding each side, so Done width = (scrollContent inner) − 72. Switched Close to `alignSelf: 'stretch'` + `marginHorizontal: 36`, which produces the same final pixel width centered.

### Glow — from 3-layer halo to smooth Gaussian-style fade

Mike saw the prior glow as three distinct concentric borders rather than a soft shadow. Replaced the hand-tuned `haloInner/Mid/Outer` with `GLOW_LAYERS`, a generated array of 14 layers each 1px further out with exponential opacity decay (peak ~0.32 at the bar edge, decaying through `0.32 * exp(-i/4.2)`). Visually merges into a continuous fade on both iOS and Android. iOS `shadowColor`/`shadowRadius` kept on the bar itself for the extra blur on iOS.

## AI Enrichment — Phase 1 pipeline shipped to DB

Built the first AI enrichment pipeline end-to-end (fetch → LLM → match → persist), mirroring the broadcast-discovery pattern. Per `docs/areas/ai-enrichment.md` Phase 1.

### What shipped

- **Sources (promotion-aware):**
  - `services/aiEnrichment/fetchUFCEventPreview.ts` — puppeteer-extra + stealth against ufc.com (JA3-protected). Browser-reuse pattern mirroring `scrapeUFCAthleteHeadshot.ts`.
  - `services/aiEnrichment/fetchTapologyEventPreview.ts` — plain fetch+cheerio for non-UFC events (tapology.com isn't JA3-protected).
  - `services/aiEnrichment/fetchEditorialPreviews.ts` — Brave search across 9-outlet allowlist (mmafighting, mmajunkie, bloodyelbow, sherdog, espn, mmamania, mmaweekly, cbssports, bjpenn). De-dupes to one article per domain, takes top N.
- **Extract:** `services/aiEnrichment/extractFightEnrichment.ts` — Claude Haiku 4.5 with structured-output JSON, prompt caching on the system prompt. Per-fight records: `redFighter`, `blueFighter`, `weightClass`, `cardSection`, `isMainEvent`, `rankings`, `odds`, `whyCare`, `stakes[]`, `storylines[]`, `styleTags[]`, `pace`, `riskTier`, `confidence`. **Hard rule against fabrication** — fields the source doesn't support stay empty.
- **Schema:** added six nullable fields to `Fight` (`aiTags JSONB`, `aiPreviewShort`, `aiPreview`, `aiEnrichedAt`, `aiSourceUrls TEXT[]`, `aiConfidence`). Migration `20260516000000_add_ai_fight_enrichment_fields` applied to Render.
- **Persist:** `services/aiEnrichment/persist.ts` — fuzzy-match LLM records against **UPCOMING fights only** (skip CANCELLED) with surname-anchored pair-agnostic scoring. Upserts straight to DB; no admin review inbox (per Mike's call — soft narrative, confidence floor + source URLs are sufficient audit).
- **CLI:** `scripts/enrich-event.ts` — `--event-id <id>` / `--next` / `--next-ufc` with `--persist` (default dry-run). Also `scripts/dump-ufc-event-preview.ts` for raw-text inspection.
- **`searchBrave.ts`** — added optional `freshness` param (`pd|pw|pm|py`). Backward-compatible.

### Dev event: MVP MMA 1 — Rousey vs. Carano (2026-05-16)

Initially started against the next UFC event (Allen vs. Costa, May 16) and got a structural backbone but zero narrative — UFC.com card page has no editorial content and MMA Fighting hadn't posted a preview yet. Switched dev to MVP Rousey vs. Carano on Mike's suggestion. Order-of-magnitude better signal — every editorial outlet had multi-paragraph coverage.

Result on Rousey card (49 DB rows: **11 UPCOMING, 38 CANCELLED** — the CANCELLED rows were old Tapology-imported matchups; persist correctly ignored them):

- 4 sources fetched (Tapology + ESPN + CBS Sports + BloodyElbow), ~31KB total text
- LLM extracted 9 fights, all with grounded narrative (100% coverage on covered subset)
- 9/9 LLM records matched DB rows at score ≥1.90/2.00 (most perfect 2.00)
- 2 undercard fights (Aline Pereira vs. Jade Masson-Wong, Chris Avila vs. Brandon Jenkins) had no editorial coverage → correctly left empty
- Cost: $0.023 / event, 13.6s wall time
- Tokens: 11,996 input / 2,208 output (no cache hits — single-event run)

Sample written `aiPreviewShort`: *"Two pioneering women's MMA legends meet for the first time after a combined 27 years away from competition in a historic Netflix main event."*

### Decisions

1. **No admin review inbox.** Broadcast discovery uses one because broadcaster facts are wrong/right; AI narrative is soft. Confidence floor + `aiSourceUrls[]` provenance is enough. (If quality slips later, we can add review without restructuring.)
2. **Trigger cadence: three passes per event** — T-10 days, T-5 days, T-2 days. Logic: deeper editorial coverage drops as the event approaches; a 10-day-out pass gets baseline structure, T-5 gets the bulk of preview articles, T-2 catches fight-week stories and late replacements. Each pass overwrites the prior `aiTags`/`aiPreviewShort`. Implementation deferred.
3. **Match only against `fightStatus = UPCOMING`.** Stale CANCELLED rows are a real footgun — Rousey card had 38 of them. Surfaced by Mike's reality check on the 49-fight count.
4. **Promotion-aware source dispatch.** UFC → ufc.com (Puppeteer) + editorial Brave. Everything else → Tapology page + editorial Brave. Single code path through `extractFightEnrichment`.
5. **LLM honesty rule load-bearing.** Empty arrays / null fields are valid output. Don't render below a confidence floor on the consumer side.

### Cost shape

At $0.023/event × 3 passes × ~6 events/week = ~$0.40/week ≈ **$21/year**. Well under the area doc's $300/year ceiling, leaves room for Phase 2+.

### Open / next session

- **Cron infrastructure.** Need a job that scans for events at T-10/T-5/T-2 days and runs `enrich-event.ts` against each. Mirror the broadcast-discovery GH Actions cron pattern.
- **Idempotency / re-run policy.** Three passes will overwrite freely; the `aiEnrichedAt` timestamp captures which pass produced the current row.
- **Render targets undecided.** Schema is in place but no surface yet. Most likely first surfaces (per area doc Tier 1): "Why care" snippet on the fight card, stakes box on the rating screen, full preview on SEO web pages.
- **Backfill once-shot for the next 2–3 big upcoming cards** could be useful for picking a render surface — we'd have real data to design against.
- **Coverage shortfall on thin UFC cards** (Allen vs. Costa, Song vs. Figueiredo) is a known limitation, not a bug. The pipeline returns structured backbone only when sources are absent. Not worth fixing until the trigger cadence is in place and we have a real sample.
