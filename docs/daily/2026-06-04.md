# 2026-06-04 ‚Äî Web app Home screen

## What shipped

Built a dedicated **Home screen** for the web app (`goodfights.app`), mirroring the
mobile home. It is now the default landing page at `/`. Previously `/` *was* the
upcoming-events list and `/events/upcoming` redirected to it.

### New Home sections (in order)
1. **Editorial band** ‚Äî `EditorialHero` + `EditorialSecondary` (the blog). This is
   now the **only** place on web that carries the editorial band.
2. **This Weekend** ‚Äî upcoming events between now and end of week (Mon ‚Üí full next
   week), compact teaser cards. (`WeekendEventsSection`)
3. **Hot Fights** ‚Äî most-hyped upcoming bouts (`/community/top-upcoming-fights`).
4. **Recent Good Fights** ‚Äî top community-rated fights, last month (`top-recent-fights`).
5. **Classic Good Fights** ‚Äî top-rated fights 3+ years old (`/community/classic-fights`).
6. **Top Comments** ‚Äî most-upvoted recent reviews (`/community/top-comments`).
7. **Classic Comments** ‚Äî the `throwback` review from the same endpoint.
8. **Highlighted Fighter** ‚Äî daily-rotating AI-enriched fighter, big portrait + bio
   summary + their top-rated fight (`/community/highlighted-fighter`, NEW).
9. "More from the blog" link ‚Üí `/blog`.

**No external news section** on web (that aggregator stays mobile-only by design).

### Routing / nav
- `/` ‚Üí new `HomeClient` (kept the live-event one-shot redirect to `/events/live`).
- `/events/upcoming` ‚Üí now a real Upcoming Events page (was a redirect to `/`).
- Navbar: added a **Home** tab; **Upcoming** now points to `/events/upcoming`;
  `isActive('/')` is exact-match again.

### Editorial removed from tabs
`EditorialHero` / `EditorialSecondary` removed from **Live**, **Past**,
**Good Fights**, and the new **Upcoming** page ‚Äî Home only.

## Backend
Added `GET /api/community/highlighted-fighter` (`routes/community.ts`): picks from
fighters with a confident AI bio (`aiProfileConfidence >= 0.5`) + a portrait,
rotates by UTC day, and attaches their top-rated completed fight. Returns
`{ data: { fighter, topFight } }` or `{ data: null }`. Auto-deploys from `main`
(backend buildFilter only ignores web/mobile/docs).
