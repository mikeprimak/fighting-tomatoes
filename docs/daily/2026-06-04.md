# 2026-06-04 — Web app Home screen

## What shipped

Built a dedicated **Home screen** for the web app (`goodfights.app`), mirroring the
mobile home. It is now the default landing page at `/`. Previously `/` *was* the
upcoming-events list and `/events/upcoming` redirected to it.

### New Home sections (in order)
1. **Editorial band** — `EditorialHero` + `EditorialSecondary` (the blog). This is
   now the **only** place on web that carries the editorial band.
2. **This Weekend** — upcoming events between now and end of week (Mon → full next
   week), compact teaser cards. (`WeekendEventsSection`)
3. **Hot Fights** — most-hyped upcoming bouts (`/community/top-upcoming-fights`).
4. **Recent Good Fights** — top community-rated fights, last month (`top-recent-fights`).
5. **Classic Good Fights** — top-rated fights 3+ years old (`/community/classic-fights`).
6. **Top Comments** — most-upvoted recent reviews (`/community/top-comments`).
7. **Classic Comments** — the `throwback` review from the same endpoint.
8. **Highlighted Fighter** — daily-rotating AI-enriched fighter, big portrait + bio
   summary + their top-rated fight (`/community/highlighted-fighter`, NEW).
9. "More from the blog" link → `/blog`.

**No external news section** on web (that aggregator stays mobile-only by design).

### Routing / nav
- `/` → new `HomeClient` (kept the live-event one-shot redirect to `/events/live`).
- `/events/upcoming` → now a real Upcoming Events page (was a redirect to `/`).
- Navbar: added a **Home** tab; **Upcoming** now points to `/events/upcoming`;
  `isActive('/')` is exact-match again.

### Editorial removed from tabs
`EditorialHero` / `EditorialSecondary` removed from **Live**, **Past**,
**Good Fights**, and the new **Upcoming** page — Home only.

## Backend
Added `GET /api/community/highlighted-fighter` (`routes/community.ts`): picks from
fighters with a confident AI bio (`aiProfileConfidence >= 0.5`) + a portrait,
rotates by UTC day, and attaches their top-rated completed fight. Returns
`{ data: { fighter, topFight } }` or `{ data: null }`. Auto-deploys from `main`
(backend buildFilter only ignores web/mobile/docs).

## Verification
- `tsc --noEmit` clean on web; `tsc -p tsconfig.production.json --noEmit` clean on backend.
- `next build` clean — `/` and `/events/upcoming` both render as expected.
- **Not yet tested live.** Home's Highlighted Fighter band stays hidden until the
  backend deploy lands (web fails closed → section returns null on 404).

# 2026-06-04 (evening) — Web Home "This Weekend" + "Hyped Upcoming Fights" polish

Remote-control session reworking two web home bands. All shipped to `main`
(auto-deploys `packages/web` → goodfights.app). Commits `7612764`, `757e5f1`,
`b97a523`, `6c6b071`, `795ea01`, `31cf9f6`, `87b93d2` (plus a few squashed style
commits in between).

## Home is org-agnostic now
The home screen no longer applies the org filter. `WeekendEventsSection` was the
only home band that did; removed it. The org filter governs **only** the Live /
Upcoming / Past / Good Fights tabs. (See [[lesson_promotion_hide_three_layers]] —
this is a *user* org filter, distinct from promotion hiding.)

## "This Weekend" cards redesigned
`packages/web/src/components/home/WeekendEventsSection.tsx`:
- **Single column** (was 2-col grid); banner image is **full-bleed** to the card's
  left edge, fills full card height, wider on desktop (`w-32 sm:w-48`).
- **Uniform card height** `h-32`; text block **top-aligned** (`items-start`),
  chevron stays `self-center`.
- **Larger text**: promotion label `text-sm`, event name `text-base`. Date/time
  stays `text-[11px]`.
- **Promotion underscores stripped** for display (`TOP_RANK` → `TOP RANK`) via a
  local `promotionLabel()`.
- **First-fight start time in ET** next to the date — earliest known bell
  (`earlyPrelimStartTime` → `prelimStartTime` → `mainStartTime`), formatted in
  `America/New_York` with an `ET` suffix.
- **"Top Fights:" list** replaces the old "N fights" count: up to 3 bouts with
  **avg hype > 7 AND ≥ 3 hype ratings**, ordered by hype desc. Grey label, inline,
  comma-separated, `line-clamp-2`.
- **Perf**: `getEvents` limit dropped **30 → 16**. This band was the slowest to
  load (above the fold) because `includeFields` aggregates hype for every fight on
  every event returned; upcoming events are soonest-first so 16 still covers a full
  week. (Q from Mike: "is there a load order?" — no, all home bands fetch in
  parallel via independent React Query; This Weekend just *finished* last due to
  query weight, not ordering.)

## "Hyped Upcoming Fights" (was "Hot Fights")
`packages/web/src/components/home/FightSections.tsx` +
`packages/web/src/components/fight-cards/UpcomingFightCard.tsx`:
- Renamed the band **"Hot Fights" → "Hyped Upcoming Fights"**.
- **Grouped by event** with a per-event subheading like **"UFC in 2 days" /
  "RAF in 2 weeks"** (`promotionLabel` + `relativeEventTime`, calendar-day based).
  Grouping preserves the API's hype/band order (soonest-leading).
- `UpcomingFightCard` gained a `showEvent` prop + event name/date line (mirrors
  `CompletedFightCard`); briefly used, then **dropped within this section** since
  the group header now carries event context.

## Classic Good Fights → "See all"
`packages/web/src/app/fights/top/page.tsx` + `SectionHeading`:
- Added a **"See all"** link on the Classic Good Fights band →
  `/fights/top?period=all`.
- Good Fights page now reads its initial `period` from the URL
  (Suspense-wrapped `useSearchParams`, validated against `TIME_PERIODS`, defaults
  to `month`).
