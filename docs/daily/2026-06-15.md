# 2026-06-15

## "Event Last Night" home section (web + mobile)

Added a new section above the "events this weekend" per-day list on both home
screens that surfaces the most recent **UFC** event (UFC only — not other
promotions) on the day(s) right after it ran. Shows UFC Freedom 250 now.

- **Web:** `packages/web/src/components/home/WeekendEventsSection.tsx`
  - New `['home','last-night-ufc']` query (`type: 'past'`, `limit: 8`, no fights).
  - `lastNightUFC` filter: `promotion === 'UFC'` and the event's UTC calendar day
    (`eventDayKey(e.date)`) is today or yesterday (`daysSince` 0–1).
  - Renders an `Event Last Night` `SectionHeading` + reused `EventDayCard` above
    the weekend day sections. Early `return null` now also accounts for it.
- **Mobile:** `packages/mobile/app/(tabs)/home.tsx`
  - New `['events','past','lastNightUFC']` query, same filter logic, reuses the
    existing `EventRow` + `Section`.

**Why the today-or-yesterday window:** UFC `date` is a noon-UTC placeholder for
the ET calendar date; UFC Freedom 250 = `2026-06-14T12:00Z` (eventDayKey Jun 14),
`mainStartTime` `2026-06-15T00:00Z` (8pm ET Sat). Cards start late and roll past
midnight ET, so "yesterday" catches the common Sat-night → Sun-morning case.
`daysSince <= 1` means the section naturally disappears two days later.

Both packages `tsc --noEmit` clean (no new home.tsx errors). Committed +
pushed on `claude/user-focused-pivot-l8l6mg` (3d873f24). Not deployed — this is
the long-lived feature branch (no main merge, so no Vercel auto-deploy / OTA).
