### Sector Swell Monitor — designed

Mike asked for a recurring program (modeled on the existing weekly Broadcast Discovery job) to scan industry resources monthly and produce a written briefing on the selling landscape.

- **Design plan written:** `docs/plans/sector-swell-monitor-2026-05-10.md`
- **Architecture:** mirrors `packages/backend/src/services/broadcastDiscovery/` — GH Actions cron → Brave Search → Claude Haiku per-result extraction → Sonnet briefing synthesis → markdown report committed to repo + email via Resend.
- **Cadence:** monthly, 1st of month at 09:00 UTC.
- **Cost:** ~$0.80/month (~$10/year).
- **Build estimate:** 6–8 hours of focused work, phased.

### Prototype briefing — written

Built a real prototype manually, not a mock. Pulled current signals across all five dimensions via web search, wrote it as the format we want.

- **File:** `docs/sector-monitor/2026-05.md`
- **First draft was prose-heavy** with inline links — Mike rejected it ("too difficult to read with all the links involved... too high reading level... assume i am very tired and have a lot on my plate").
- **Second draft is the keeper.** Scannable bulletin format. Top half reads in 60s, full bottom in ~3 min.
- **Required sections in order:** Bottom line → What you should do → Why (things that got better) → Why (things that didn't) → What changed since last month → Watch table (3 items, next 60 days) → Appendix (per-dimension scores + sources + methodology).
- **No inline links in the body.** Sources block only at bottom.
- **The cron design plan was updated to lock in this format.** Future months must match the May 2026 prototype's shape exactly.
