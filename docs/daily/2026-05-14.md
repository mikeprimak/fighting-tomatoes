## Marketing infra — landing-page install-referrer forwarding (Session #4)

Continued straight into the install-attribution half of the same deadline. Commit `6d363f2`.

### What shipped
- New `packages/landing/attribution.js` — IIFE script that runs on `DOMContentLoaded`, reads `utm_*` from `location.search`, and rewrites every `apps.apple.com` and `play.google.com` `<a>` on the page:
  - Play Store: appends `&referrer=<urlencoded utm string>` — surfaced to the installed app on first launch via Google Play Install Referrer API. **This is real install attribution on Android.**
  - App Store: appends `?ct=<utm_campaign or utm_source>` (max 100 chars) — surfaced in App Store Connect Analytics > Sources. **This is real install attribution on iOS within Apple's ecosystem.**
- Wired into `index.html` and `download.html` via `<script src="/attribution.js" defer></script>`
- Auto-deployed via push to `main` (Vercel project `fighting-tomatoes-backend`, despite the misleading name — that's the project serving the static landing site at goodfights.app)

### Verified live
- `curl https://goodfights.app/attribution.js` → script served
- `curl "https://goodfights.app/?utm_source=fightingtomatoes&..."` → has `<script src="/attribution.js" defer>` in HTML
- `curl -L https://goodfights.app/download` → same
- Browser verification (Mike's homework): hit https://goodfights.app/?utm_source=test&utm_campaign=test, inspect store buttons, confirm `?ct=test` (Apple) and `&referrer=utm_source%3Dtest...` (Google) on hrefs

### Known gap
- `packages/landing/vercel.json` has UA-based server redirects from `/download` → store. Mobile users hitting `/download` directly bypass the JS, so any `?utm_*` on `/download` URL is forwarded literally and ignored by the stores (they expect `ct=` and `&referrer=` formats, not raw `utm_*`). For paid/social campaigns linking to `/download`, this leaks attribution.
- Mitigation options for later: (a) edge function reading query + UA, building the right destination dynamically; (b) replace the redirect with a tiny JS-based redirect page. ~1-2hr work. Lower priority since legacy traffic enters at `/`.

- Legacy `fightingtomatoes.com` Sunday-morning installs from here on out arrive at the App/Play Store with attribution attached. Future Sunday spikes are diagnosable instead of mysterious.
