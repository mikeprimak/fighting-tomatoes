## Shipped — `packages/mobile/app/(tabs)/home.tsx`

Event cards iterated based on Mike's feedback:

- **Event card infinite-height bug fixed** — image was `height:100%` under an
  auto-height (stretch-derived) parent; absolute-fill the image so the card height
  comes from the text column. (`cc842bd`)
- **Card blurb = main event's AI preview** — replaced the hyped-fight list, then the
  most-hyped-fight preview, with the **main event** fight's `aiPreviewShort`,
  confidence-gated ≥ 0.5. Loads upcoming events with `includeFights`. (`25630ec`, `04a0d8c`)
- **Main-event detection fix** — `orderOnCard: 1` is the headliner (top of bill), not
  the highest orderOnCard. Was picking an un-enriched prelim → blank cards (e.g. UFC
  "Muhammad vs Bonfim", which sits at order 1 with a good 0.85-confidence preview).
  Now picks the **lowest** orderOnCard. (`91cab95`)
  - Note for later: web `EventCard` "up next" uses *highest* orderOnCard and calls it
    "last to walk out" — looks inverted vs this data convention. Out of scope today.
- **Start time + timezone + channel** — card meta line shows the main event start time
  in the user's local zone with tz abbr, then the region broadcast channel
  (e.g. "8:00 PM ET · ESPN+"), via `formatEventTime`/`getTimezoneAbbreviation` +
  `useEventBroadcasts`. (`91cab95`, `298ac3b`)
- **Section titles** now read **"Events Today / Events Tomorrow / Events <weekday>"**.
- **Date moved to a section subline** under each day heading (e.g. "June 7th"); removed
  from individual cards. Org-logo overlay removed from the card image.
- **Description** now shows in full (no line clamp) at a smaller size (11/15). (`ae3b3c9`)
