# 2026-05-23

## Web app sidebar overhaul — one long session

Mike was looking at the web app on his phone and started flagging small UX
issues on the right-rail sidebar. By the end of the session the entire
"About you" sidebar had been redesigned: new blocks added, old ones
restructured, mobile layout fixed, persistence wired up, and a new `/fan-dna`
route shipped.

All changes touch `packages/web/src/components/sidebar/` (eight blocks) and
the main feed page (`packages/web/src/app/page.tsx`) + `/profile` +
`/followed-fighters` + a new `/fan-dna` route.

### Final sidebar order (top to bottom)

1. **IdentityBlock** — avatar, name, personality type, stats (Ratings/Hype/Comments)
2. **FanDNABlock** — "Your Fan DNA" — top 4 trait cards (headline + body explanation + family-colored stat chip) + "See full DNA" link
3. **DistributionBlock**
4. **YourCommentsBlock** *(new)* — daily-rotating spotlight on one of the user's comments. Variants: "Your top comment" (when ≥1 upvote) and "You wrote this recently". Label tells you *why* it surfaced (no ambiguity).
5. **FollowedFightersStrip**
6. **UpcomingHypedBlock** — header: "Upcoming fights you're hyped for" (flame icon, heatmap-colored badge per fight)
7. **SpotlightBlock** — "A good fight you might love" (past-fight pick only; rotation removed). Star icon next to fighter names; event line is `Event: <promotion> · <event name>`.
8. **MightLikeBlock** — "Upcoming fight you might like" — single card, rotates through 10 recommendations every 10 seconds with a 250ms fade

### Block-level changes worth flagging

- **Heatmap badges replace flat yellow chips** on UpcomingHyped and Spotlight — flame/star icons colored by `getHypeHeatmapColor`, white number overlaid with text-shadow. Matches `CompletedFightCard` / `UpcomingFightCard` style.
- **SpotlightBlock simplified** — dropped the daily rotation between `past-fight-pick`, `hottest-take`, `taste-callout`. Always surfaces the past-fight pick now (hidden if no candidate ≥ 7 avg rating). Less variety, but the variant labels were confusing alongside other sidebar copy.
- **MightLikeBlock** — changed from a 5-fight list to a single fight per slot that rotates every 10s. Pool fetches 10 candidates.
- **YourCommentsBlock** — new variant-rotating spotlight (top + recent). Discussed UX tradeoffs with Mike before building: "mix of recent + top" was rejected because users can't tell which they're looking at; went with single-card-labeled-by-why. Reply-count variant deferred until backend tracks reply counts.

### `/fan-dna` page (new route)

Linked from the FanDNABlock "See full DNA" button. Personality hero card +
all trait cards grouped by family color (affinity/behaviour/prediction/identity).
Modeled on the mobile `app/activity/fan-dna.tsx` screen.

### Mobile layout fix (the original issue)

On phone-sized viewports, the right sidebar got stranded below the
infinite-scroll feed and was unreachable. Resolution:

- `<lg`: IdentityBlock + FanDNABlock + "More about you →" link hoisted above the feed. Rest of sidebar hidden.
- `lg+`: unchanged.
- `/profile`: now renders the full sidebar content (in the same order) + the existing Edit/Activity/Followed Fighters links, so the "More about you" link delivers everything that was hidden.

### Followed-fighters page → 2-column

- Main column = user's followed fighters
- Sidebar = new `PopularFightersBlock` (top-followed fighters platform-wide; quick-follow button; fighters the user already follows are filtered out)
- 0-0-0 records hidden — placeholder data, not real signal

### Org filter persistence (parity with mobile)

`packages/web/src/lib/orgFilter.tsx` now persists `selectedOrgs` to
`localStorage['events_org_filter']` (same key as mobile's AsyncStorage),
restored on mount with a `hydrated` guard to avoid overwriting the saved
selection on first render.

### Vercel auto-deploy flakiness

Two commits in this session got skipped by Vercel's auto-deploy webhook
(`b56639f`, `3916d2e`). Pattern: commits arriving 3–4 minutes after a prior
push get debounced. Cumulative state still lands on prod via the next
successful build, so end-state is correct — but the user can't see their
change in isolation.

Workaround: empty commit re-triggers the webhook. Not a bug worth chasing
unless it becomes regular — log it here so future-me knows it's a known
quirk, not a real config break.
