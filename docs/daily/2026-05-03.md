## Changes Made

### Backend: Gamebred parser — drop name OR-fallback in event lookup
- **Bug**: The May 1 Gamebred Bareknuckle MMA event in the app showed Anthony Smith vs Chase Sherman, who actually fought on the April 10 Gamebred event. Several other fights similarly misplaced.
- **Root cause**: Tapology hadn't yet appended a "vs. X" headliner suffix to either upcoming event, so events 140031 (Apr 10) and 140032 (May 1) both rendered with the title "Gamebred Bareknuckle MMA". The parser's lookup used `OR: [{ufcUrl}, {promotion + name}]` — when the second event was scraped, the name branch matched the first event's row and merged its fights into one card. ufcUrl-by-itself would have routed each event to its own row.
- **Fix**: `gamebredDataParser.ts` now looks up events by `ufcUrl` only. The Tapology event ID is the unique stable identifier; the name fallback was redundant and caused the merge.
- File: `packages/backend/src/services/gamebredDataParser.ts:183-190`

### Backend: Data repair script + run
- New: `packages/backend/src/scripts/fixGamebredEventCollision.ts`
- Re-fetches the Tapology Gamebred hub + each event page (cheerio, no puppeteer), ensures one Event row exists per ufcUrl, then re-points fight.eventId for any fight currently sitting on the wrong row. User ratings stay attached to the fight rows so they survive the move.
- DRY_RUN=1 supported. Idempotent — re-running with no remaining mismatches is a no-op.
- **Run against prod**: created the missing Apr 10 (140031) row, moved 10 fights from the merged May 1 row to it (incl. Smith vs Sherman, Cooper III vs Eliziario, Pena vs Alexandre, Petrov vs Sosa, Polanco vs Schnurpel, Rosa vs Sojo, Santos vs Inocente, Santos vs Quinones, Soriano vs Ramos, Sowma vs Kovalenko). 117 fights already on the correct row. 2 ambiguous-pair skips (Cade Howell vs Christopher Wingate appears on both Gamebred Boxing 3 and 4 — they fought twice).

The bug only triggers when two upcoming events on the same hub share an exact title — uncommon for promos that always include a "vs." suffix. Not patched in this PR; tracked as a follow-up so the broader sweep is its own commit.

### What landed

- `packages/backend/src/services/rafLiveParser.ts` — `parseRAFLiveData` now takes a 3rd optional `BackfillOptions` arg. Five flags plumbed (mirrors BKFC):
  - `nullOnlyResults` — RAF parser already gates winner/method writes on `!dbFight.winner` / `!dbFight.method`, so this is a no-op for safety/consistency
  - `skipStaleLiveReset` — gates the "lifecycle-premature COMPLETED → UPCOMING" downgrade
  - `skipNotifications` — gates the `notifyNextFight()` call on COMPLETED transitions
  - `skipCancellationCheck` — gates the entire CANCELLED↔UPCOMING reconciliation pass
  - `completionMethodOverride` — when set, stamps `Fight.completionMethod` + `Fight.completedAt` on COMPLETED transitions (audit trail)
  - Header log line now reads `[RAF BACKFILL]` vs `[RAF PARSER]` based on whether any flag is set
- `packages/backend/src/services/backfillRAFResults.ts` — new wrapper. Clone of `backfillBKFCResults.ts` shape (child-process JS scraper → JSON output → parser). Path differences: `scrapeRAFLiveEvent.js`, `live-event-data/raf/`, `raf-live-` filename prefix. Stamps `completionMethod = 'backfill-raf'`.
- `packages/backend/src/scripts/backfillResults.ts` — added `case 'raf':` in the dispatcher, import, and updated the header coverage comment.

### Note on RAF auto-publish behavior

RAF is not in `DEFAULT_PRODUCTION_SCRAPERS` (`['ufc', 'oktagon', 'bkfc', 'onefc', 'pfl']`), so `buildTrackerUpdateData` writes only to shadow fields (`trackerWinner`, `trackerMethod`, etc.) — same as the live tracker. The orchestrator's candidate query filters on the published `winner` field, which means RAF events with shadow-only data will keep showing up as candidates each day until an admin publishes. That's idempotent (each rerun produces the same shadow values) but a touch wasteful. If RAF publish becomes a recurring annoyance, the right fix is either to add RAF to `DEFAULT_PRODUCTION_SCRAPERS` or to have the backfill query also exclude rows where the shadow already matches.

## UFC historic backfill — full sweep architecture

Built and shipped the deep historic backfill the morning's `HANDOFF-historic-sweep-2026-05-03.md` flagged. Two distinct passes (results + headshots), distinct sources, distinct safety contracts.

### Scope discovery (the real numbers)

The handoff doc framed UFC results as "the highest-value pass." DB queries inverted that:

- **6,597 of 6,816 UFC fights (97%) have null winners.** 1993–2025 events were imported as event/fight/fighter skeletons with no result data — only 2026 has winners (recent live scraper).
- Worst gaps: 2023/2024/2025 are total blackouts (winner+method+round all null on every fight).
- Per-org orchestrator's previous count of "7 events / 18 fights" missed all of this because most historic UFC events have `scraperType=null`.
- Headshot side: 1,470 UFC-affiliated fighters missing `profileImage` (much larger than the earlier 57-fighter scope, which required scraperType='ufc').

### Source choice: ufcstats.com (results) + ufc.com athlete pages (headshots)

Confirmed with user: ufcstats.com is the canonical UFC stats site, single-page event list (770+ events back to UFC 2 / 1994), every event-detail page has full winner/method/round/time inline. Wikipedia rejected as messier per-era table parsing.

For headshots: `ufc.com/athlete/<slug>` `<meta property="og:image">` is the canonical URL. Verified across active (Conor, Islam) AND deeply-retired (Royce Gracie, Tito Ortiz) fighters — all return correct headshots. Deeply-retired pre-2010 fighters (e.g. Tank Abbott) have a ufc.com page but no og:image; Tapology fallback flagged for v2.

### Files added (commit `2b6dd94`)

- **`services/scrapeUFCStatsHistoric.ts`** — fetch+cheerio; two functions: list (filters out the leading next-upcoming row by date) and event-detail (extracts winner flag, fighter URLs, method, methodDetail, round, time, weight class from the inline fight table).
- **`services/scrapeUFCAthleteHeadshot.ts`** — shells to curl (axios + Node OpenSSL TLS gets 403 from ufc.com bot protection; curl's Schannel fingerprint passes). Extracts og:image, returns ok/no-page/no-image/error.
- **`scripts/backfillUFCHistoric.ts`** — orchestrator. Date-based event matching with name-token jaccard tiebreaker. Fighter-pair matching: Pass 1 exact normalized, Pass 2 token-overlap fuzzy fallback (≥0.5 jaccard, top-result must lead second by 0.1+ to avoid ambiguity). Null-only writes; `completionMethod='backfill-ufcstats'`. Per-year + total stats. Env: BACKFILL_UFC_YEARS, BACKFILL_UFC_LIMIT, BACKFILL_UFC_DRY_RUN, BACKFILL_UFC_RATE_LIMIT_MS.
- **`scripts/backfillUFCHeadshots.ts`** — finds UFC-affiliated fighters (eventscraperType='ufc' OR name `^UFC`) with null profileImage. Slug priority: existing `Fighter.ufcAthleteSlug` → name-derived. Successful derived slugs are written back (organic slug-discovery). Uploads via existing `uploadFighterImage` → R2.
- **`.github/workflows/ufc-historic-backfill.yml`** — manual dispatch only. Inputs: mode (results|headshots|both), years CSV, event_limit, headshot_limit, dry_run. 6h timeout.

### Smoke-test results

- **Dry-run, 3 events of 2025**: 33/40 fights matched (83%) without fuzzy. After adding the token-overlap fallback: **36/40 (90%)**. Residual: nickname swaps ("King Green" / "Bobby Green") and uncatalogued late-additions ("Nurullo Aliev vs Shem Rock"). The fuzzy pass caught middle-name drops ("Ian Machado Garry" → "Ian Garry") and surname-only matches.
- **Live-write, 1 event (UFC Fight Night Royval vs. Kape, 2025-12-13)**: 11 fights persisted with winner + "KO/TKO (Kick)" / "U-DEC" / "SUB (Anaconda Choke)" methods, R+time, all stamped `backfill-ufcstats`. Verified by re-querying the 11 rows. Royval vs Kape correctly recorded as Kape KO/TKO (Punches) R1 3:18.
- **Headshot dry-run, 5 candidates**: 2 succeeded via ufc.com (1 derived slug, 1 existing), 3 returned `no-image` (Tank Abbott, Danny Abbadi, Daiche Abe — old fighters with no og:image). Confirms the ~40% ufc.com hit rate for the long-tail.
