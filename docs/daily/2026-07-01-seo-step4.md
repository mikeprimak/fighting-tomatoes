# 2026-07-01 — Programmatic SEO step 4: /fighters hub + division hubs + Person JSON-LD

**SHIPPED + verified live same session** (commits `379c6d0b` backend, `630a86cc` web).
Deployed in two ordered pushes — backend first, verified live on Render, THEN web —
to avoid the ISR-bakes-empty race that bit step 3 (see `2026-07-01-seo-step3.md`).

## Why this step

The ~1,100 fighter deep pages were in the sitemap (step 3) but were **orphans** —
zero internal links. Hubs are the internal-link crawl graph; sitemap discovery
alone ranks poorly. Fighter template SSR turned out to be mostly done already
(client component's initial render SSRs the About/record/h1 — verified in prod
HTML), so the real gaps were JSON-LD + hubs + nav.

## What shipped

**Backend** (`379c6d0b`)
- `/api/fighters` grew `indexable=true` (filters by the shared SEO gate —
  `fighterIndexWhere()` from `lib/seoIndex.ts`, same whitelist as the sitemap),
  `weightClass=<enum>` (unknown value → empty result, not a Prisma 500),
  `sort=ratings|name`; list select now includes slug/isChampion/championshipTitle/
  averageRating/totalRatings.
- `/api/fighters/divisions` — indexable-fighter counts per weightClass (Fastify
  static segment beats `:id`, so no route conflict).

**Data fix (applied to prod)** — `scripts/backfillFighterRatingAggregates.ts`:
`Fighter.totalRatings / averageRating / totalFights / greatFights` were **all
dead** (0 across 9,946 rows — same disease as `Event.totalRatings`,
`lesson_dataset_aggregates_dishonest`). Recomputed from live per-fight data
(weighted avg; greatFights = rated fights ≥8.5). 7,575 rows updated; 4,307 now
non-zero. Top: Gaethje 1,135/9.16, Poirier 1,127/8.85, McGregor 1,038/8.49.
Dry-run default; **monthly re-run logged in `docs/operations/maintenance.md`**
(nothing maintains these at rating time — they drift).

**Web** (`630a86cc`)
- `/fighters` hub: SSR, most-rated-first, 48/page crawlable `<a>` pagination,
  division chips (only divisions with ≥`MIN_DIVISION_COUNT`(3) indexable fighters).
- `/fighters/division/[division]`: facet hub; slug↔enum is a pure mechanical
  transform (`lib/divisions.ts`) so it can't drift from the schema; malformed
  segment → 404; thin division → renders but `noindex,follow`.
- Fighter detail: `Person` JSON-LD (name/alternateName/image + confidence-gated
  tldr as description).
- `fighters/sitemap.xml` now leads with the 11 division-hub URLs; root sitemap +
  navbar gained `/fighters`.
