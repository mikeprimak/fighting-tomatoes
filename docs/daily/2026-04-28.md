## Changes Made
### Mobile: timezone abbreviation in section start times
- `packages/mobile/utils/dateFormatters.ts` — added `getTimezoneAbbreviation(date: Date)` helper. Uses `Intl.DateTimeFormat` with `timeZoneName: 'short'` to extract the device's local TZ name (e.g. `EDT`, `PST`); falls back to a `GMT±H` offset if Intl can't produce a short name (older Hermes builds).
- `packages/mobile/app/(tabs)/events/index.tsx` — updated the local `formatTime` alias (used by Main Card / Prelims / Early Prelims section headers) to append the timezone, so each displayed start time now reads e.g. `8pm EDT` instead of `8pm`.

## Notes
- DST is handled automatically because `Intl.DateTimeFormat.formatToParts(date)` is called per-date — same code returns `EST` in winter and `EDT` in summer.
- Web upcoming events page (`packages/web/src/app/page.tsx` + `EventCard.tsx`) doesn't currently surface section start times, so no change was needed there. If we add per-section times to the web `EventCard` in the future, mirror the helper in `packages/web/src/utils/dateFormatters.ts`.
