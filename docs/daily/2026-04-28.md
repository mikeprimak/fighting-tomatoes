## Symptoms
User reported the upcoming UFC event (UFC Fight Night Della Maddalena vs Prates, May 2 2026, Perth) has the Steve Erceg vs Tim Elliott bout listed twice.

## Root Cause (recap from 4/25 + 4/28)
Same pattern as the Davey Grant vs Juan/Adrian Luna Martinetti dup from 4/25:

1. UFC corrects a fighter's display name on ufc.com.
2. Pre-`47c74a6` daily UFC scraper used `firstName_lastName` as the upsert key — so a renamed fighter forked a brand-new `Fighter` row instead of updating the original.
3. The forked `Fighter` row implies a fresh `(eventId, fighter1Id, fighter2Id)` tuple, so the unique constraint doesn't fire and a second `Fight` row gets written.
4. Result: two fights on the same event with the same name signature but different fighter ids.

`47c74a6` (this morning) fixed the upstream cause by switching the UFC fighter upsert key to `ufcAthleteSlug`, but pre-fix duplicate rows already on Render are still there. `06efa7c` (4/25) fixed the live-tracker's un-cancellation so duplicates stay cancelled instead of resurrecting on every 5-min tick — also a runtime guard, doesn't clean DB rows.

## Changes Made
### Backend: `src/scripts/dedupeUpcomingUfcFights.ts`
One-shot dedupe script for upcoming UFC events. Detects fight pairs that share a name signature on the same event but have different fighter row ids. For each duplicate group, picks the canonical row to keep using:

1. Both fighters have `ufcAthleteSlug` set (post-`47c74a6` row),
2. More user activity (ratings + reviews + predictions + tags),
3. Older `createdAt`.

Deletes the loser inside a transaction along with its dependent rows (FightRating/Review/PreFightComment/FightPrediction/FightTag have `onDelete: Cascade` already; CrewMessage/Prediction/RoundVote/Reaction don't, so the script clears them explicitly). Stale duplicate `Fighter` rows are intentionally left for `scripts/fighter-dedup/merge-fighters.ts` to consolidate, since that one handles ratings/follows/aliases properly.

```bash
# Render-connected env (DATABASE_URL points at the External URL)
cd packages/backend
npx ts-node src/scripts/dedupeUpcomingUfcFights.ts            # dry run, lists groups
npx ts-node src/scripts/dedupeUpcomingUfcFights.ts --apply    # delete losers
npx ts-node src/scripts/dedupeUpcomingUfcFights.ts --event <id> --apply
```
