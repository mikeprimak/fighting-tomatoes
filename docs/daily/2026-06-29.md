## Logo PNG → editable SVG (brand asset, no app/DB changes)

Mike's `GOOD-FIGHTS-LONG-LOGO.png` was LLM-generated raster; he wanted it as an editable
SVG. The pre-existing `GOOD-FIGHTS-LONG-LOGO.svg` in the repo is a **fake** — 0 vector
paths, just 13 base64 PNGs wrapped in SVG (that's why it was never editable). Built a real
vectorization, iterated several times on his feedback. **Pure local image work — touched no
app code, no DB, no deploys. All originals left intact (he said keep everything).**

### The key lesson (load-bearing for any future logo/trace work)
**Tracing the pixels of an AI-generated logo always looks "hand-drawn/wavy"** because the
wobbles are baked into the raster. Filled shapes (GOOD) hide it; thin outlines (FIGHTS)
expose every wobble. Smoothing (median + gaussian before potrace) cleans broad raster
defects (e.g. a real bump+notch in the "D" top edge — confirmed at pixel level, not a trace
artifact) but never gets truly crisp. **The actual fix Mike steered to: stop tracing, and
re-typeset the words in a REAL font** — clean vector glyphs, mathematically straight.

### Tooling (all pip, Windows, no cairo/inkscape/potrace binaries needed)
- `potracer` (pure-Python potrace) — **gotcha: pass a `bool` array, NOT `uint32`** (uint32 is
  read as packed bitmap words → garbage). It also emits the full image frame as `path[0]` →
  skip any subpath whose bbox spans the whole canvas, else even-odd inverts the fill.
- `resvg_py` (`svg_to_bytes`) to rasterize SVG for visual QA (cairosvg fails — no libcairo on
  Windows). `skimage.skeletonize` + `sknw` for the centerline experiments.
- `fontTools` (`TTFont` + `SVGPathPen` + `TransformPen`) to extract real glyph outlines as SVG
  paths and bake an affine (scale/flip-Y/rotate/translate) into the coords → self-contained
  paths, **no font dependency at render time**, stroke-width in intuitive px.
- Generator script kept in scratchpad (`final_typeset.py`); not committed (one-off asset gen).

### Iterations (what Mike rejected → why)
1. Single-color alpha-trace as one fill (IoU 0.986 vs original) — faithful but not editable per-piece.
2. FIGHTS as centerline **strokes** (skeleton) so stroke-width = thickness knob — "scribbly/toddler".
3. Smoothing-spline on centerlines — fixed scribble but organic/wavy, not geometric.
4. FIGHTS as clean filled-outline (median+gaussian, like GOOD) — clean but no stroke knob.
5. Corner-aware centerline (straight L between hard corners, curve only on gentle bends) —
   straighter, strokeable, but still not crisp (AI source is the ceiling).
6. **FINAL: re-typeset both words in a real font.** Asked Mike via AskUserQuestion → he chose
   **Barlow Condensed Bold Italic** (closest-fitting system font; Montserrat too wide/round).

### Final deliverable: `GOOD-FIGHTS-LOGO-typeset.svg`
- 3 editable groups: `id="hand"` (traced fill, light smoothing keeps knuckle detail),
  `id="good"` (Barlow glyphs, **filled**), `id="fights"` (Barlow glyphs, **stroked** →
  `stroke-width="15"` in px on the group is the bold knob).
- Both words same font = guaranteed consistent. Glyphs **horizontally widened ~1.23×** +
  tight tracking (0.015) + small word gap (0.06) per Mike's last note ("wider chars, less
  space"). Laid out on a ~−6° rising italic baseline, vertically centered, fit inside the
  2799×738 canvas with right margin (right edge ~2717).
- **Not a pixel match** to the AI original (it's a clean real-font interpretation in the same
  athletic-italic spirit) — that's the deliberate trade for "clean". Trivial knobs if revisited:
  widen factor, tracking, stroke weight, overall size, baseline slant.
