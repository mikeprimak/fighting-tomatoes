# 2026-06-18

## Blog fight cards — embeddable `CompletedFightCard` for articles

Built a read-only fight card component for embedding live community ratings inside blog posts, starting with the draft "Every Conor McGregor Fight, Ranked".

### The constraint
Blog posts render markdown → HTML via `marked`, injected with `dangerouslySetInnerHTML` (`packages/web/src/app/blog/[slug]/page.tsx`). React components **cannot** be inlined in markdown. The existing `TweetEmbeds` solves this with a placeholder-hydration pattern, which we reused.

### What shipped (3 files + article)
- **`packages/web/src/components/BlogFightCard.tsx`** — presentational card. Two fighters (avatar + first/last name via `FighterAvatar`), event name + date, and a prominent aggregate-rating square (heatmap-colored via `getHypeHeatmapColor`) with the rating count. Optional `#N` rank badge. Whole card is a `<Link href="/fights/[id]">` (acquisition CTA). Spoiler-neutral by design — no winner/method — so it's safe in any post.
- **`packages/web/src/components/BlogFightCards.tsx`** — client hydrator. Finds every `<div class="gf-fight-card" data-fight-id="…" data-rank="…">` in the post body, fetches each fight via the public `getFight(id)` (`optionalAuth`, works logged-out), and `createPortal`s a `BlogFightCard` into each placeholder. Bad/failed IDs are dropped silently so a typo never breaks the article.
- **`page.tsx`** — render `<BlogFightCards />` right after `<TweetEmbeds />`.
- **`2026-06-17-conor-mcgregor-fights-ranked.md`** — inserted all 15 placeholders under their ranking headings with real prod fight IDs (queried read-only).

### Authoring syntax
Drop this on its own line (blank lines around it so `marked` passes the raw `<div>` through — verified it preserves `class` + `data-*`):
```html
<div class="gf-fight-card" data-fight-id="FIGHT_UUID" data-rank="1"></div>
```
`data-rank` is optional. Grab `FIGHT_UUID` from the fight's URL on goodfights.app (`/fights/<id>`).

### Verified
- `tsc --noEmit` clean on `packages/web`.
- `marked.parse` preserves the placeholder div + attributes.
- `GET /fights/:id` returns `event {name,date}`, `fighter1/2 {firstName,lastName,profileImage}`, `averageRating`, `totalRatings` (same shape `CompletedFightCard` consumes).

## Life-like admin draft preview (replaces inline HTML preview)

Admin's old draft "Preview" dumped backend-rendered markdown HTML into a modal — no web-app CSS, no React, so embedded fight cards (and any future component) never rendered. Replaced it with a real web-app preview opened in a new tab.

- **`packages/web/src/app/blog-preview/[slug]/page.tsx`** — new `force-dynamic`, `noindex/nofollow` route that renders ANY post (drafts included) through the real web app. Shows an amber "Preview — unlisted" banner.
- **`packages/web/src/components/BlogArticle.tsx`** — extracted the article body (header, hero, blog-content, TweetEmbeds, BlogFightCards, share, tags) from `/blog/[slug]/page.tsx` so live + preview render identically. Live page keeps its SEO/JSON-LD and now renders `<BlogArticle post={post} />`.
- **`posts.ts`** — `getPost(slug, { includeDrafts: true })` bypasses the prod draft gate (for the preview route only).
- **`admin.html`** — `previewDraft(slug)` now `window.open('https://goodfights.app/blog-preview/<slug>')` instead of the inline modal. (Old `draftPreviewModal` markup left in place, now unused/harmless.)

**Access decision (Mike, 2026-06-18):** preview links are **unlisted + noindex, NOT token-gated**. Drafts are viewable by anyone with the exact `/blog-preview/<slug>` URL — hidden from search/sitemap/RSS/index, just not secret. Chose simplicity over the prior "drafts fully hidden in prod" guarantee.

**Caveat:** preview shows the *deployed* web app, so push to main (Vercel auto-deploys) before previewing brand-new edits. Verified: `tsc` clean + `pnpm build` succeeds, `/blog-preview/[slug]` present in the build manifest.
