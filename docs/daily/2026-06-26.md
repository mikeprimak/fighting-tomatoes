# 2026-06-26 — Affiliate monetization for "How to Watch" links

**Session goal (Mike):** "Monetize my app — set up affiliate links for all the How-to-Watch links for each org per country/region, web + mobile."

## What shipped (code on `main` working tree; NOT yet committed/deployed)

The How-to-Watch system already had a `BroadcastChannel.affiliateUrl` field and link precedence that preferred it — the plumbing was 80% there. This session built the **tracked-redirect + click telemetry + operator tooling** around it.

### 1. Click tracking (owned data)
- New Prisma model **`BroadcastClick`** (`broadcast_clicks` table) — one row per outbound broadcaster-link tap: channel, event, region, card section, tier, platform (web/mobile), `targetUrl`, `isAffiliate` (monetized vs homepage fallback), `ipHash` (sha256[:16], not PII), UA, referer.
- Migration `20260626000000_add_broadcast_clicks` — **hand-authored SQL, applied to prod via `migrate deploy`** (status was clean, only this migration pending). Table is live on Render.

### 2. Tracked redirect endpoint
- **`GET /api/r/b?c=<channelId>&e=&r=&s=&t=&p=&b=`** (`routes/broadcasts.ts`). Resolves the real target server-side (eventDeepLink → affiliateUrl → homepageUrl), logs a `BroadcastClick`, then 302s out. **No open-redirect** — destination is never carried in the URL, always re-resolved from `channelId`, so updating an affiliate URL takes effect immediately for all clients. Logging failures never block the redirect.
- `broadcasts.ts` now rewrites every `deepLink` in the API response into a tracked `/api/r/b` URL (via `buildTrackedLink`). **Web + mobile needed zero render changes** — they already open `entry.deepLink`.

### 3. Platform tagging
- `x-client-platform: web` / `mobile` header added to both api clients' central request fns (web `lib/api.ts`, mobile `services/api.ts`); added to CORS `allowedHeaders`; read in the broadcasts route and baked into the tracked link as `p`. Lets us segment click revenue web vs mobile.

### 4. Admin panel (web-based, `public/admin.html`, Broadcasts tab)
- **Affiliate URL** field added to the channel editor (the lever). Channel list shows a 💰 "Affiliate: Yes/No" column + "X of Y channels monetized" summary.
- New **"💰 How-to-Watch Click Revenue"** panel: total / monetized / homepage-lost clicks, by region, by platform, per-channel click table with an "Add link" shortcut for un-monetized channels. Backed by new `GET /admin/broadcast-clicks/stats?days=`.
- **Always-available reference card** at the top of the Broadcasts tab (collapsible `<details>`): how the monetization works, the lever, "do this in order" signup steps, programs-by-region table, the reality-check caveat, compliance note. So the operator never has to leave the panel to know what to do.

### 5. Docs
- `docs/marketing/affiliate-programs-by-region.md` — full research doc (networks, per-region programs, signup links, compliance, wiring). Researched 2026-06-26 via web search.

## Key findings (affiliate landscape)
- **Join Impact.com first** (Fubo, Kayo AU, many US streamers), then **Awin** (Sky UK + EU), then **DAZN direct** (best combat-sports channel globally), **Fubo** (~$30/sub via Impact), **Amazon Associates** (Prime Video).
- **Reality check:** ESPN+ (UFC US PPV), TNT Sports (UFC UK), UFC Fight Pass have **no open affiliate program**. Monetizable surface = aggregators (Fubo/Sling/Kayo/Sky) + DAZN. Supplemental revenue at ~300 users, not primary.
