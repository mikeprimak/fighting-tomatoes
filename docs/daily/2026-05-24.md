# 2026-05-24

## Blog system added to the web app (markdown-file based)

Mike wanted to publish an article previewing the RAF wrestling card
(Khamzat Chimaev vs Dillon Danis). There was no blog infrastructure in
either `packages/web` or `packages/landing`, so we built a minimal
markdown-file blog into the Next.js web app and dropped the first post in
as a draft.

### How it works

Posts are plain `.md` files in `packages/web/src/content/posts/` with
gray-matter frontmatter. No CMS, no DB ‚Äî files are read at build/request
time on the server.

Frontmatter shape:

```yaml
title, slug, date (YYYY-MM-DD), author, excerpt, tags[], draft (bool)
```

- `src/lib/posts.ts` ‚Äî reads the posts dir, parses frontmatter
  (`gray-matter`), renders body to HTML (`marked`). **Drafts are included
  in dev but stripped when `NODE_ENV === 'production'`** ‚Äî so a
  `draft: true` post is previewable locally / on `next dev` but never
  ships to prod (both the listing and a direct `/blog/<slug>` URL 404 in
  prod via `notFound()`).
- `src/app/blog/page.tsx` ‚Äî index, lists posts newest-first, shows a
  "Draft" badge.
- `src/app/blog/[slug]/page.tsx` ‚Äî post detail. `generateStaticParams`
  + `generateMetadata`. Shows a "hidden in production" notice on drafts.
  Uses async `params: Promise<{slug}>` per Next 16.2 conventions.
- Rendered markdown styled via a `.blog-content` block in `globals.css`
  (no `@tailwindcss/typography` plugin installed; hand-rolled prose
  styles using the existing theme vars).
- Navbar: added "Blog" link (Newspaper icon), desktop + mobile.
- `sitemap.ts`: added `/blog` + maps published (non-draft) posts.

New deps in `packages/web`: `gray-matter` ^4, `marked` ^18.

## Aside: live-event app-icon badge question (no code)

Mike asked whether a red dot could show on the home-screen icon while
events are live. Answered: yes, via the app icon badge
(`expo-notifications` `setBadgeCountAsync`), but since the app is closed
it must be driven server-side through the push payload (`badge` field,
ideally a silent content-available push). Reliable on iOS; Android
badges are launcher-dependent and flaky. Existing Expo push pipeline
(`backend/src/services/notificationService.ts`, per-user `User.pushToken`)
makes it feasible. Scoping only ‚Äî nothing built. Mike dropped it.
