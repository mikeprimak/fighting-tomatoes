## Fixed: PFL San Diego not live-tracking (missed early-card start time)

Remote-control, mid-event. PFL San Diego 2026 (https://pflmma.com/event/pfl-san-diego-2026)
had started for real â€” the Weems fight (order #5) had just ended â€” but the app still showed
the event UPCOMING and was not live-tracking it.

**Root cause (two layers).** The event had only `mainStartTime` set; `prelimStartTime` and
`earlyPrelimStartTime` were both null and every fight was labeled `cardType: "Main Card"`.
`eventLifecycle.ts` Step 1 flips UPCOMING â†’ LIVE off
`getStartTime() = earlyPrelimStartTime || prelimStartTime || mainStartTime || date`, so the
earliest time it knew was the **main card** (8 PM PT) â€” hours after the real **early card**
(8 PM ET / 5 PM PT). It stayed UPCOMING and the PFL VPS tracker was never dispatched. Two
scraper bugs produced the missing/wrong times:

1. **Early-card time never captured.** pflmma.com renders `Early Card: 8:00 PM ET` /
   `Main Card: 10:00 PM ET` with **no pipe delimiter**, but the body-text fallback regexes in
   `scrapeAllPFLData.js` only matched a pipe-delimited `LABEL | TIME`. So `prelimStartTime`
   (PFL "Early Card" == prelims) stayed null. (The `.card-date` header path also yielded
   nothing here.)
2. **Main-card time stored an hour late.** `parsePFLEventStartTime()` in `pflDataParser.ts`
   hardcoded ET = UTC-5 (`hours + 5`) â€” correct only in winter. In EDT it stored the 10 PM ET
   main card at **03:00Z instead of 02:00Z**.

**Immediate fix (prod DB).** Set `prelimStartTime=2026-06-28T00:00:00Z` (8 PM ET),
`mainStartTime=2026-06-28T02:00:00Z` (corrected), `startTimeSource='manual'`,
`startTimeConfidence=1.0`. Within ~1 min the prod lifecycle flipped it LIVE and the VPS PFL
tracker began advancing the card â€” early-card fights #5â€“#10 (incl. Weems) COMPLETED with
tracker data, main-card #1â€“#4 correctly still UPCOMING. Confirmed working.

**Code fixes (root cause; affect future PFL events).**
- `pflDataParser.ts` `parsePFLEventStartTime()` Priority-2 path: replaced `+5` with DST-aware
  `eventTimeToUTC(eventDate, eventStartTime, 'America/New_York')` (the helper the prelim path
  already used). tsc clean.
- `scrapeAllPFLData.js` early/prelim fallback: added no-pipe variants
  (`\bEARLY\s*CARD\b[^\n]{0,40}?(TIME)` + prelim + early-prelim), bounded to the same line so
  they don't grab the main-card time. Verified against 4 page-text formats incl. the failing
  `Early Card: 8:00 PM ET`.

**Deploy note:** did NOT redeploy the backend mid-live-event (redeploys rotate tokens / log
users out, per `lesson_render_redeploy_token_rotation_logouts`). The scraper fixes only matter
for future events, so they ride the next normal backend deploy. **Changes are uncommitted on
`main`** (`pflDataParser.ts`, `scrapeAllPFLData.js`).
