# 2026-06-13

## Tapology Cloudflare → Scrapfly (the real fix)

Continuation of the 2026-06-11/12 investigation. Root cause confirmed earlier:
Tapology escalated Cloudflare from JA3/TLS fingerprinting to a JS challenge that
escalates to an **interactive Turnstile widget on datacenter IPs** (GitHub
Actions runners + the Hetzner VPS). No headless browser solves Turnstile, so
every Tapology daily scrape / live track / backfill broke at once. The
real-Chrome+stealth path (`tapologyBrowser.js`) only clears it from a
**residential** IP — fixes local dev, not CI/VPS.

Decision (Mike): go with **Scrapfly** Web Unlocker, **test for one month**.

### Shipped to `main`

- **`5c93c85b` feat(scrapers): Scrapfly fallback for Tapology Cloudflare Turnstile**
  - Env-gated Scrapfly path added to `packages/backend/src/services/tapologyBrowser.js`.
    When `SCRAPFLY_KEY` is set, `launchTapologyBrowser` wraps the browser so every
    `page.goto()` to a `tapology.com` URL is served by Scrapfly (`asp=true` solves
    Turnstile server-side on a residential pool, `render_js=true`) + `page.setContent`,
    with an injected `<base href="https://www.tapology.com/">` so the scrapers'
    `link.href` reads resolve absolute (page URL is `about:blank` under setContent).
  - **Transparent — zero consumer edits.** All 8 `scrape*Tapology.js`, the live
    tracker (`tapologyLiveScraper.ts`), and `backfillTapologyResults.ts` route
    through this module. `SCRAPFLY_KEY` unset → original real-Chrome path runs.
  - Built on top of the `fix/tapology-cloudflare` real-Chrome work, which was
    merged to main in the same push (it was based on pre-shelving main; one
    conflict in `backfillTapologyResults.ts` import block, resolved to keep both
    the shelving helpers and the `fetchTapologyHtml` import).
- **`4f140fc9` ci(scrapers): inject SCRAPFLY_KEY into Tapology workflow envs** —
  added `SCRAPFLY_KEY: ${{ secrets.SCRAPFLY_KEY }}` to the env block of all 10
  Tapology-fetching workflows (4 active daily + 4 shelved for un-shelve
  readiness + `tapology-backfill` + `tapology-live-tracker`).
