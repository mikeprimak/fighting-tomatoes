### Root cause of the 83-record gap: cross-window overlap, not missing persistence

Counted unique fightIds across all 40 files:

```
total records:    1000
unique fightIds:   917
overlap (dupes):    83
```

The 10 windows' candidate files weren't perfectly disjoint because the picker filters dynamically (`WHERE aiTags IS NULL ORDER BY rating count`). When the first windows started writing, the ordering shifted under later windows' picks, so some fights appeared in two windows' candidate sets.

This is exactly the race-safety footnote in `docs/HANDOFF-historic-enrichment-multi-window-2026-05-18.md`:
> "if Window A is mid-batch when Window B picks, B's offset is computed against the DB state at B's pick time — so by the time A's batch hits the DB, B might end up with some overlap. The writer's UPSERT is idempotent and the second write is harmless"

So 1022 = 105 (pre-campaign) + 917 (unique writes from 10 windows). Campaign is **fully closed at this chunk**.
