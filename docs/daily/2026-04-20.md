Needed a clean screen-capture for a promo video where UFC 328: Chimaev vs. Strickland (2026-05-09) appears as the first event on the Upcoming tab, ahead of sooner-dated cards. Scoped the override to one account so no other user is affected.

**Change:** `packages/backend/src/routes/index.ts`, inside the `type === 'upcoming'` branch of `/api/events`. When the authenticated request's `user.email === 'avocadomike@hotmail.com'`, the hardcoded event ID `b992560c-fe62-417b-8953-e3323cdf7b2a` is moved to position 0 of the sorted lite list **before** pagination, so it lands on page 1 of the infinite-scroll feed. ### Fix "index" header flash on app launch
Noticed the word "index" briefly appearing at the top of the screen on app open before the home screen loaded. Root cause: `app/(tabs)/index.tsx` is a redirect-only route that renders an `ActivityIndicator` while `useHasLiveEventWithLoading` resolves, then `<Redirect>`s to `/live-events` or `/events`. The tabs navigator rendered its header during that loading window, and because the `Tabs.Screen` for `index` set `href: null` but no `headerTitle`, the header defaulted to the route name — literally `"index"`.

**Change:** `packages/mobile/components/TabBar.tsx:163` — added `headerShown: false` to the `index` screen options.

JS-only, so it can ship via `eas update --branch production` without a new native build.
