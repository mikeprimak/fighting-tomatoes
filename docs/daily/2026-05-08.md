## Work shipped

### Migration: `20260508000000_event_broadcast_unique_with_section`

The `EventBroadcast` unique constraint was `(eventId, channelId, region)` ‚Äî that prevented the same channel from carrying different card sections at different tiers (e.g. Sportsnet Sub on prelims + Sportsnet PPV on main card in CA). Migration drops the old constraint, drops the underlying old unique INDEX (Postgres `DROP CONSTRAINT` doesn't always drop the index when it predates the constraint), and adds a new `UNIQUE NULLS NOT DISTINCT (eventId, channelId, region, cardSection)`.

Postgres 17 on Render ‚Äî `NULLS NOT DISTINCT` supported.

Audit of all 94 `PromotionBroadcastDefault` rows. Only one default was flatly wrong against current sources:
