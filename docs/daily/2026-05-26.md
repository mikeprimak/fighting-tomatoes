### DBX duplication + stuck status (data repair, prod via Render)

QA surfaced a fight in My Activity rendering as an upcoming card that the user
hadn't (knowingly) rated. Investigation found two systemic DBX problems:

1. **Duplication** — the DBX 5 scrape (2026-02-14) over-pulled DBX 3's entire
   card: 13 duplicate bouts on "DBX 5: Clark vs. Bittencourt" that belong to
   "DBX 3: Rozenstruik vs. Perez". Canonical = the DBX 3 copy. Idempotent
   migration (dry-run first): for each dup, ratings/predictions/reviews/comments
   migrated to canonical (per-user conflicts resolved by keeping canonical's),
   aggregates recomputed, then the DBX 5 dup deleted. **13 dups merged, 0 user
   ratings lost.** DBX 5 went 29 → 16 fights (its real card).
2. **Stuck status** — nearly every fight on the 7 COMPLETED DBX events was still
   `UPCOMING` (the tapology path never transitions DBX fights). **83 fights
   flipped UPCOMING → COMPLETED.**

Verified after: 0 remaining cross-event dup pairs, 0 stuck UPCOMING/LIVE on
completed DBX events. Throwaway inspect/fix scripts deleted (no code committed).

**Why this survived the big 2026-05-23 phantom cleanup:** that sweep's Pass 2
only deleted pairs on **5+ Tapology events** (rematch-safety threshold). DBX
dupes were on only 2 events → below threshold → survived. Today's repair fills
that gap with the same principles.
