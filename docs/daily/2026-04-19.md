## Google Sign-In broken on BOTH platforms — root-caused + shipped

User reported that after yesterday's "fix", Android was now also broken ("Google sign-in failed: Invalid Google token"). iOS still broken. Investigated and found two compounding bugs from yesterday.

### Bug 1: Backend code never committed

Yesterday's doc described a `getGoogleAudiences()` helper that parses `GOOGLE_CLIENT_ID` as a comma-separated list. The Render env var was updated to a comma-joined string of old + new webClientIds. But the code change was only in the working tree — never committed or pushed. Render was still running the pre-change code, which passed the raw env var string to `verifyIdToken({ audience })` as a single audience. No real idToken's `aud` claim equals the literal string `"A,B"`, so **every** sign-in failed verification. This is why Android — which had "worked by luck" before (audience validation was silently skipped while `GOOGLE_CLIENT_ID` was unset) — started failing: setting the env var enabled the check, but the deployed code couldn't parse the list.

### Bug 2: iOS client ID was never in the env var

Yesterday's env var held `<new-webClientId>,<old-webClientId>`. On iOS, `@react-native-google-signin/google-signin` issues idTokens with `aud = iosClientId`, not the webClientId. So even after fixing Bug 1, iOS would still fail — the iOS audience wasn't in the allow-list at all.

### Fix shipped this session

| Step | Change |
|---|---|
| 1 | Committed the staged-but-unpushed `getGoogleAudiences()` changes in `authController.ts` + `auth.fastify.ts`. Commit `207c86c`, pushed to `main`, Render auto-deployed. |
| 2 | User updated Render env var `GOOGLE_CLIENT_ID` to a three-value comma-separated list: `<new-webClientId>,<new-iosClientId>,<old-webClientId>`. Selected **Save, rebuild, and deploy** to apply both the new commit and env var in one swap. |

Verified via background poller on the `rndr-id` response header — old instance `c52df29f-a0ba-4955` swapped to `007e5bc1-27bd-4736` within ~16 min. `/api/auth/google` returned a clean 400 `INVALID_TOKEN` for a junk test body (endpoint healthy).

User confirmed **Sign in with Google works on both iOS and Android** post-deploy.
