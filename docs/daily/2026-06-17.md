## What happened

### GA4 general analysis (28d, lifetime — GA started ~June 6)
- 776 sessions / 703 users, but **~half is bot/redirect junk** (Singapore/

  Direct 200, Reddit 31 (55% engaged). ChatGPT now appears as a referral source.
- Real humans ≈ 300, not 703. Bounce 79%, avg session 85s (both distorted by bots).
- Blog posts pull real organic readers (McGregor/Holloway, Gaethje tribute, etc.).

### `ga.js` recovered
- `packages/backend/scripts/ga.js` had been lost (never committed). Recreated it
  and **added an `organic` report** = real organic-search sessions per month
  (the buyer-credible web number; excludes bot/redirect traffic via GA channel
  grouping). Also **gitignored `ga-service-account.json`** — it was untracked but
  NOT ignored (secret-leak risk; the memory claiming it was ignored was wrong —
  corrected in `reference_ga4_query_script`... TODO if needed).

### Sale-value framing (Mike led)
- Mike pushed back on the "web traffic doesn't matter to a buyer" stance. He's
  right — traffic properties do sell; value tracks quality + durability + whether
  it's monetized. Of his traffic, the **organic-search slice** is the part with
  real traffic-sale value.
- Added 3 posture principles to `docs/areas/sale-value.md` (his words): doc is
  flexible/not gospel, **web traffic matters** (separately-sellable asset), any
  value-improving opportunity matters regardless of stage.

### Shipped: web metric in admin Acquisition tab + monthly email
- `services/googleAnalytics.ts` — backend GA4 client. Creds from
  `GA_SERVICE_ACCOUNT_JSON` (prod) or local `ga-service-account.json` (dev).
  **Null-safe**: no-ops if unconfigured, never blocks the snapshot.
- `routes/adminStats.ts` — `web` section on `/api/admin/metrics/acquisition-snapshot`
  (this-month / last-month organic sessions + MoM growth + months-of-history) and
  `POST /api/admin/metrics/web-report/test` to send the email on demand.
- `utils/email.ts` — `EmailService.sendWebTrafficReport` (operator-facing).
- `services/backgroundJobs.ts` — cron `0 9 1 * *` monthly web-traffic email;
  no-ops until creds land. Recipient via `WEB_REPORT_EMAIL` / first `ADMIN_EMAILS`.
- `public/admin.html` — "Web property (organic search)" section + "Send test email".
- Verified locally: GA module returns June = 139 organic sessions / 118 users /
  85 engaged. Production tsconfig typecheck clean (only pre-existing untracked
  Tapology probe scripts error, unrelated).
