# Session 3 - Daily odds graph system + admin draft viewer

## Summary
Mike's two asks while reviewing the McGregor drafts: (1) a way to READ drafts
(no easy local way), and (2) a system that checks the main-event odds daily,
auto-updates the article, and renders a day-to-day odds-change graph. Built both.

## Draft reading
- Drafts are hidden in prod everywhere (`includeDrafts = NODE_ENV !== 'production'`
  in `packages/web/src/lib/posts.ts` AND `routes/editorial.ts`). In DEV they render.
- Immediate unblock: started `packages/web` dev server (localhost:3000); the 3
  draft slugs return 200 and render in-context. Gave Mike the links.
- Durable: built an admin-panel draft viewer (below).

## Odds system (main-event line only, per Mike's scope)
- **Data:** `packages/web/src/content/odds/ufc-329.json` - American moneyline per
  fighter, one snapshot per date. Seeded with REAL dated lines: May 16 open
  (Holloway -310 / McGregor +250, DraftKings), May 20 (Kalshi -270/+245), Jun 17
  (consensus mid -400/+300). No fabrication.
- **Graph:** `scripts/odds/render-odds-graph.mjs` -> self-contained dark-theme SVG
  line chart of IMPLIED WIN PROBABILITY, written to
  `public/blog/ufc-329-odds-graph.svg`. Zero deps. Embedded in the full-card
  article under a question H2 ("How have the McGregor vs Holloway odds moved?")
  with a `<!--ODDS-SNAPSHOT-->...<!--/ODDS-SNAPSHOT-->` block the updater rewrites.
- **Daily updater:** `scripts/odds/update-odds.mjs` - fetches The Odds API free
  tier (h2h/MMA, consensus = median implied prob across US books), appends/dedupes
  today's snapshot, regenerates SVG, rewrites the snapshot block + `updated:` date.
  No `ODDS_API_KEY` or fetch failure = harmless no-op (never invents a point).
  No-ops after eventDate+1.
- **Automation:** `.github/workflows/ufc-329-odds.yml` - daily 17:00 UTC cron,
  `permissions: contents: write`, runs updater, commits+pushes to main if changed.
  Needs repo secret `ODDS_API_KEY` (Mike signs up free at the-odds-api.com).
- Tested: no-key no-op, mock-fetch full pipeline (median math correct), and
  article snapshot-block + updated-date rewrite (all against throwaway copies).
