# 2026-06-05 — Home screen redesign (mobile)

Branch: `home-screen-redesign` (off `main`, pushed). **Not** on `winner-predictions` — that's
separate, paused work.

## Shipped (commit eefeaca)

`packages/mobile/app/(tabs)/home.tsx`:

1. **Per-day event sections.** "Events This Weekend" is gone as a single block. The
   weekend events are now grouped by calendar day, each day rendered as its own
   `Section` titled **Today / Tomorrow / weekday name** (e.g. "Saturday"). New
   module helper `eventDayLabel()` derives the label from the event's UTC calendar
   day vs the user's local today (Event.date is a UTC-hour placeholder). "See all"
   shows only on the first day's section.
2. **Full-width horizontal event cards.** Replaced the 2-col `EventThumbnail` grid
   with `EventRow`: image on the left (fixed 120px, full-height), name + date +
   top fights on the right. Old `eventGrid`/`eventThumb*` styles removed; new
   `eventList`/`eventRow*` styles added. Dead `isEventToday` helper removed.
3. **Top fights inside each card.** Each card lists its genuinely-hyped matchups as
   plain "A vs B" text (max 4). Source = the already-loaded `/top-upcoming-fights`
   data, keyed by event id (`hypedByEvent` map). That endpoint already enforces the
   exact floor Mike asked for — `MIN_HYPE_COUNT=3` AND `MIN_AGGREGATE_HYPE=7` — so
   **no new endpoint was needed**.
4. **"The Latest" moved** from the top of the feed to just below the Highlighted
   Fighter section (kept, not removed — per Mike's correction).

Type-check: home.tsx clean (0 new errors; repo baseline unchanged).
