## First-fight notification ping now fires for ALL orgs (was manual-tracker only)

**Context:** BKFC Nashville tonight (9pm ET). Mike asked whether the event-wide
start time is understood correctly so undercard-fight followers get notified.

**Investigation findings:**
- BKFC Nashville: `mainStartTime = 2026-06-20T01:00Z` (9pm ET), `prelimStartTime`/
  `earlyPrelimStartTime` null. Mike confirmed the undercard actually starts at 9pm,
  so 9pm IS the whole-event start. `getStartTime()` returns mainStartTime â†’ event
  flips UPCOMINGâ†’LIVE at 9pm and the VPS BKFC tracker is dispatched then. No
  too-late-start problem.
- Notification model has two intended rules:
  1. **Each fight pings when it's "up next"** â€” fires when the PRIOR fight
     completes (`notifyNextFight` â†’ `notifyFightStartViaRules`). âś… Works on every
     org.
  2. **The opener pings at event UPCOMINGâ†’LIVE** (nothing precedes it). This code
     existed in `eventLifecycle.ts` Step 1 but was gated behind
     `if (event.useManualLiveTracker)`.
- **The bug:** `useManualLiveTracker = false` for **all 67** upcoming/live events
  (every UFC/BKFC/ONE/PFL/Oktagon/Tapology/RAF). So the opener ping ran for
  **nobody**. The opener's followers got nothing until the second fight started.
- Incidental wrinkle: ONE/PFL/Sherdog parsers ALSO ping a fight on UPCOMINGâ†’LIVE
  (the fight going live), which accidentally covered their opener. UFC, BKFC,
  Oktagon, RAF, and reliable-tracker Tapology (e.g. Zuffa) had no opener coverage
  at all.

**Fix (`packages/backend/src/services/eventLifecycle.ts`, commit dc844dd6):**
Removed the `useManualLiveTracker` gate so the first-fight ping fires for every
event at the UPCOMINGâ†’LIVE transition. Picks the opener via
`fightStatus: 'UPCOMING'` ordered by `orderOnCard: 'desc'` (skips a cancelled
opener). Fires inside Step 1 *before* the VPS tracker is triggered, so no race.

**Why it's safe (no double-notify):** `notifyFightStartViaRules` marks all matched
rows `notificationSent: true`. So the parser on-LIVE ping (ONE/PFL/Sherdog) and
the Step 1.7 section-start aggregate both find the opener already sent and skip
it.

**Deploy:** Pushed to `main` â†’ Render backend auto-deploy (change is under
`packages/backend/**`, not in `render.yaml` buildFilter ignoredPaths). Must be
