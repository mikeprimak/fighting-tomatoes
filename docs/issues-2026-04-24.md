## 1. `[x]` No-tracker event lifecycle + "Bouts" label

Events without a production live tracker now:

## 2. `[x]` RAF Wrestling win methods

RAF's site (realamericanfreestyle.com) doesn't publish a method field — only scores. Implemented inference:
- Margin ≥ 10 points → `Tech Fall (9-3)` (freestyle wrestling tech-superiority threshold)
- Smaller margin → `Decision (9-6)`
- No scores present → `Decision`

Changes:
- `rafDataParser.ts` — new `inferRAFMethod` helper + call site.
- `rafLiveParser.ts` — inline same logic so live updates use the new labels.
- Mobile `formatMethod` (`shared/utils.ts`) now pretty-prints wrestling enum values too (`PIN`, `TECH_FALL`, `TECH_SUPERIORITY`, `DISQUALIFICATION`, `FORFEIT`, `INJURY_DEFAULT`) in case a future scraper surfaces them.

**Limitation:** RAF site doesn't expose pin/forfeit/injury-default data, so those cases can't be distinguished from decisions today. Noted in code comment at `inferRAFMethod`.

## 9. `[x]` Oktagon scraper fighter images

Root cause: parser stored ephemeral GH-Actions filesystem paths (`/images/athletes/oktagon/<slug>.png`) in `Fighter.profileImage` for successfully-cropped fighters. The runner's disk disappears after the scrape, so Render served 404s.

Fix:
- New `uploadFighterImageFromFile(localPath, name)` in `imageStorage.ts` — reads the cropped PNG, uploads to R2 with a content-hash key, returns the R2 public URL.
- `oktagonDataParser.ts` now prefers R2-from-local-file, falls back to R2-from-source-URL, then to the raw Oktagon URL.
- Dedupe is hash-based so re-runs don't re-upload.
- Broken DB values get overwritten on the next scrape run (daily at 18:00 UTC).

## 10. `[x]` Live Events rating-count updates

Root cause: `CompletedFightModal.tsx:onSuccess` invalidated `['liveEvents']` — a key that doesn't exist. Live Events tab reuses the upcoming-events query key (`['upcomingEvents', …]`). So ratings posted from the Live tab never triggered a refetch and counts stayed stale.

Fix: replaced the `['liveEvents']` invalidation with `['upcomingEvents']` in both `onError` and `onSuccess`, and cleaned the same stale key out of the optimistic `updateEventsCache` loop.

## 11. `[x]` Rousey/Carano missing images on MVP MMA 1L

Root cause: Rousey, Carano, Nate Diaz, Junior dos Santos, Lorenz Larkin all have legacy slug-only Tapology URLs (no numeric ID). The scraper's image map was keyed by the numeric ID parsed out of the fighter URL → `null` key → no image.

Fix: `scrapeMVPTapology.js:347-356` — when a `<li>` has no numeric-ID match, fall back positionally: first `img[src*=headshot_images]` inside the `<li>` = fighterA, second = fighterB. Takes effect on the next daily scrape (15:15 UTC).
